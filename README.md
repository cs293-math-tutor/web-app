# Nope Moments

A teacher training web app for Stanford CS293 / EDUC 473. Teachers view real classroom "shutdown" moments, where a teacher dismisses student mathematical reasoning, and practice writing research-based alternative responses. The app computes semantic similarity in-browser against four stored alternatives grounded in pedagogical frameworks.

## Getting Started

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Data Pipeline

The app uses two source CSV files in `data-sources/`:

| File | Description |
|------|-------------|
| `confirmed_nope.csv` | 413 confirmed shutdown moments with classroom context windows |
| `generated_alternatives.csv` | 413 rows with four alternative responses per shutdown |

These are merged into a single JSON file consumed by the app:

```bash
npm run prepare-data
```

This runs `scripts/prepare-data.mjs`, which joins the two CSVs on `(OBSID, teacher_turn_idx)`, filters rows with null alternatives, strips `[teacher]` prefixes from alternative text, and parses context windows into structured turn arrays. The output is `src/data/examples.json` (380 examples, ~700KB).

**You do not need to re-run this script unless the source CSVs change.** The generated `examples.json` is checked into the repo.

## How It Works

1. **Read**: The teacher sees a classroom conversation with the shutdown moment highlighted.
2. **Write**: The teacher writes their own alternative response.
3. **Score**: The app embeds the response and the four stored alternatives using `all-MiniLM-L6-v2` (run in-browser via a Web Worker), then computes cosine similarity.
4. **Compare**: All four alternatives are revealed with framework labels, citations, and individual similarity scores.

The teacher can revise their response or move to the next example. Progress is saved to localStorage.

## Modifying and Extending

### Changing the data

Replace the CSV files in `data-sources/` and run `npm run prepare-data`. The app will pick up the new `examples.json` on the next build or dev server restart. If you add or rename columns in the CSVs, update the merge logic in `scripts/prepare-data.mjs` and the TypeScript types in `src/lib/data.ts`.

### Adding a new pedagogical framework

1. Add the new `alt_` column to `generated_alternatives.csv`.
2. In `scripts/prepare-data.mjs`, include the new column in the `cleanAlternative` step and add it to the `alternatives` object.
3. In `src/lib/data.ts`, add the new key to the `Alternatives` interface and add an entry to `FRAMEWORK_INFO` with its label, citation, and description.
4. In `src/lib/similarity.ts`, add the new alternative to the `texts` array in `computeScores` and to the `scores` object.
5. Run `npm run prepare-data` to regenerate `examples.json`.

The `AlternativesPanel` component reads from `FRAMEWORK_INFO` dynamically, so the new card will appear automatically.

### Changing the embedding model

The model is loaded in `src/workers/embedding.worker.ts`. Replace `"Xenova/all-MiniLM-L6-v2"` with any Hugging Face model that supports the `feature-extraction` pipeline. Larger models produce better embeddings but increase download size and inference time. The current model is 22MB and produces 384-dimensional embeddings.

### Adjusting the similarity score thresholds

Score colors and labels are defined in `src/components/SimilarityGauge.tsx` (functions `getScoreColor` and `getScoreLabel`) and `src/components/AlternativeCard.tsx` (function `getScoreColor`). Adjust the threshold values there.

### Modifying the practice flow

The step sequence is controlled by `src/hooks/usePracticeState.ts`. The state machine uses a `Step` type (`"context" | "write" | "similarity" | "reveal"`). To add or reorder steps, update the type definition in `src/components/StepIndicator.tsx` and the state transitions in `usePracticeState.ts`.

### Styling

The app uses Tailwind CSS v4 with custom color tokens defined in `src/app/globals.css` under `@theme inline`. The main palettes are `sand-*` (warm neutrals), `slate-blue-*` (interactive elements), and `shutdown-*` (red highlights). Fonts are Playfair Display (headings) and Source Sans 3 (body), loaded via `next/font/google` in `layout.tsx`.

### localStorage

Teacher progress is stored under the key `nope-moments-state`. The schema is defined in `src/lib/storage.ts`. To reset all progress, clear this key from the browser console:

```js
localStorage.removeItem("nope-moments-state");
```

## Deployment

Deploy to Vercel by connecting the GitHub repo. Vercel auto-detects Next.js and handles the build. No environment variables or server-side configuration is required. The embedding model is fetched at runtime from the Hugging Face CDN by the browser, not bundled in the deployment.
